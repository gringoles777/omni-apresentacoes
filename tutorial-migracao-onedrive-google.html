Migração OneDrive para Google Drive

Guia passo a passo para transferência completa de arquivos
Versão 1.0  ·  Março 2026  ·  Infraestrutura & Operações
Elaborado com base na migração real realizada em 25/03/2026

Visão Geral

Este tutorial explica como migrar todos os arquivos do OneDrive (Microsoft 365) para o Google Drive usando o rclone — uma ferramenta gratuita e open source de transferência de arquivos em nuvem.

O que este processo faz: Copia tudo do OneDrive para o Google Drive sem apagar nada. Se cair no meio, retoma de onde parou. Os arquivos do OneDrive ficam intactos até você decidir cancelar o plano Microsoft.

Pré-requisitos

  • Acesso ao portal.azure.com com conta de administrador do Microsoft 365
  • Google Workspace já contratado e domínio verificado
  • Acesso SSH ao VPS da empresa
  • VPS com Linux e rclone instalado (ver Passo 1)

PARTE 1 — Preparar o VPS

1 Conectar no VPS via SSH
ssh usuario@IP-DO-VPS
2 Instalar o rclone
curl https://rclone.org/install.sh | sudo bash

Se já estiver instalado, o script pula automaticamente.

PARTE 2 — Registrar Aplicativo no Azure

Para o rclone se autenticar no OneDrive da empresa, precisamos criar um "App Registration" no Azure. Isso é feito uma única vez e só leva 5 minutos.

1 Abrir o portal Azure

Acesse portal.azure.com com a conta de administrador.

2 Criar novo App Registration
  1. Na barra de busca no topo, digite App registrations e clique
  2. Clique em New registration
  3. No campo Name, coloque rclone
  4. Clique em Register
3 Copiar o Client ID

Na página do app criado, copie o valor de Application (client) ID. Você vai precisar dele depois.

4 Criar o Client Secret
  1. Menu esquerdo → Certificates & secrets
  2. Clique em New client secret
  3. Em Description coloque rclone
  4. Clique Add
  5. Copie o valor na coluna Value (não o Secret ID)
Atenção: O Value aparece apenas uma vez. Copie imediatamente — depois de sair da página, não aparece mais.
5 Adicionar permissões de acesso ao OneDrive
  1. Menu esquerdo → API permissions
  2. Clique Add a permission
  3. Clique Microsoft Graph
  4. Clique Delegated permissions
  5. Na busca, digite Files.ReadWrite e marque Files.ReadWrite.All
  6. Clique Add permissions
  7. Clique no botão "Grant admin consent for [empresa]" → confirme
6 Habilitar autenticação por código (Device Flow)
  1. Menu esquerdo → Authentication
  2. Role até Advanced settings
  3. Em Allow public client flows → mude para Yes
  4. Clique Save

PARTE 3 — Autenticar OneDrive

Agora vamos gerar um código de autenticação que o usuário precisa autorizar no navegador — sem instalar nada no computador pessoal.

1 Gerar o código de autenticação (no VPS)

Substitua SEU-CLIENT-ID e SEU-TENANT-ID pelos valores corretos. O tenant ID está no portal Azure → Azure Active Directory → Overview.

TENANT="SEU-TENANT-ID" CLIENT_ID="SEU-CLIENT-ID" curl -s -X POST \ "https://login.microsoftonline.com/${TENANT}/oauth2/v2.0/devicecode" \ -H "Content-Type: application/x-www-form-urlencoded" \ --data-urlencode "client_id=${CLIENT_ID}" \ --data-urlencode "scope=https://graph.microsoft.com/Files.ReadWrite.All offline_access" \ | python3 -c " import sys, json r = json.load(sys.stdin) print('URL:', r['verification_uri']) print('CÓDIGO:', r['user_code']) with open('/home/usuario/device_code.txt', 'w') as f: f.write(r['device_code']) "

O comando vai mostrar uma URL e um código de 9 caracteres.

2 Autorizar no navegador
  1. Abra https://login.microsoft.com/device no navegador
  2. Digite o código de 9 caracteres exibido no terminal
  3. Faça login com a conta Microsoft da empresa
  4. Clique em Autorizar
3 Capturar o token (no VPS)
python3 << 'EOF' import urllib.request, urllib.parse, json tenant = "SEU-TENANT-ID" client_id = "SEU-CLIENT-ID" with open('/home/usuario/device_code.txt') as f: device_code = f.read().strip() data = urllib.parse.urlencode({ "client_id": client_id, "grant_type": "urn:ietf:params:oauth:grant-type:device_code", "device_code": device_code }).encode() url = f"https://login.microsoftonline.com/{tenant}/oauth2/v2.0/token" req = urllib.request.Request(url, data=data, headers={"Content-Type": "application/x-www-form-urlencoded"}) try: with urllib.request.urlopen(req) as resp: r = json.loads(resp.read()) with open('/home/usuario/od_token.json', 'w') as f: json.dump(r, f) print("TOKEN CAPTURADO COM SUCESSO") except urllib.error.HTTPError as e: r = json.loads(e.read()) print("ERRO:", r.get('error_description','')) EOF
4 Obter o Drive ID do OneDrive
python3 << 'EOF' import urllib.request, json with open('/home/usuario/od_token.json') as f: t = json.load(f) req = urllib.request.Request( "https://graph.microsoft.com/v1.0/me/drive", headers={"Authorization": f"Bearer {t['access_token']}"} ) with urllib.request.urlopen(req) as resp: d = json.loads(resp.read()) print("DRIVE ID:", d['id']) print("TIPO:", d['driveType']) with open('/home/usuario/drive_info.json', 'w') as f: json.dump({'drive_id': d['id'], 'drive_type': d['driveType']}, f) EOF

Anote o Drive ID exibido — vai ser usado na configuração do rclone.

PARTE 4 — Autenticar Google Drive

No Mac ou computador com navegador, instale o rclone e execute o comando abaixo:

1 Instalar rclone no Mac (se não tiver)
brew install rclone
2 Autorizar acesso ao Google Drive
rclone authorize "drive"

O navegador abre automaticamente. Faça login com a conta Google Workspace da empresa e clique em Permitir. O terminal exibe um JSON com o token — copie o JSON completo.

3 Salvar o token do Google Drive no VPS

Copie o JSON exibido no terminal e salve no arquivo /home/usuario/gd_token.json no VPS.

PARTE 5 — Configurar o rclone no VPS

1 Criar o arquivo de configuração
python3 << 'EOF' import json # Carregar tokens with open('/home/usuario/od_token.json') as f: od = json.load(f) with open('/home/usuario/gd_token.json') as f: gd = json.load(f) with open('/home/usuario/drive_info.json') as f: drive = json.load(f) od_token = json.dumps({ "access_token": od['access_token'], "token_type": "Bearer", "refresh_token": od['refresh_token'], "expiry": "2020-01-01T00:00:00Z" # força refresh automático }, separators=(',', ':')) gd_token = json.dumps({ "access_token": gd['access_token'], "token_type": "Bearer", "refresh_token": gd['refresh_token'], "expiry": "2020-01-01T00:00:00Z" }, separators=(',', ':')) config = f"""[onedrive] type = onedrive client_id = SEU-CLIENT-ID client_secret = SEU-CLIENT-SECRET token = {od_token} drive_id = {drive['drive_id']} drive_type = {drive['drive_type']} [gdrive] type = drive scope = drive token = {gd_token} """ import os os.makedirs('/home/usuario/.config/rclone', exist_ok=True) with open('/home/usuario/.config/rclone/rclone.conf', 'w') as f: f.write(config) print("Configuração salva!") EOF
2 Testar as conexões
# Testar OneDrive (deve listar as pastas) rclone lsd onedrive: --max-depth 1 # Testar Google Drive (sem erros = OK) rclone ls gdrive: --max-depth 1

PARTE 6 — Iniciar a Migração

1 Rodar a migração em background
python3 -c " import subprocess proc = subprocess.Popen( ['rclone', 'copy', 'onedrive:', 'gdrive:', '--transfers', '4', '--checkers', '8', '--log-file', '/home/usuario/rclone-migration.log', '--log-level', 'INFO'], stdout=open('/home/usuario/rclone-out.log', 'w'), stderr=subprocess.STDOUT ) print('Migração iniciada. PID:', proc.pid) with open('/home/usuario/rclone.pid', 'w') as f: f.write(str(proc.pid)) "
A migração roda em background. O VPS continua trabalhando mesmo que você feche o terminal.
2 Acompanhar o progresso
# Ver últimas linhas do log tail -20 /home/usuario/rclone-migration.log # Ver processo rodando ps aux | grep rclone | grep -v grep
3 Retomar se cair

Se a migração parar por qualquer motivo, basta rodar o mesmo comando do Passo 1 novamente. O rclone pula automaticamente os arquivos já copiados.

Pós-migração

Antes de cancelar o Microsoft 365: verifique no Google Drive que todos os arquivos chegaram corretamente. Compare pelo menos as pastas principais. Só depois cancele o plano Microsoft.
  1. Acesse o Google Drive e confirme que as pastas e arquivos estão lá
  2. Verifique arquivos importantes (documentos críticos, planilhas, apresentações)
  3. Confirme com a equipe que está tudo acessível
  4. Então cancele o plano Microsoft 365

Resolução de Problemas

Erro Causa Solução
AADSTS650051 App não autorizado no tenant Criar App Registration próprio no Azure (Parte 2)
AADSTS7000218 Public client flows desabilitado Authentication → Allow public client flows → Yes
Permission denied no log Pasta de log sem permissão Usar pasta home do usuário em vez de /var/log
Invalid Credentials (Google) Token expirado Definir expiry para data passada, rclone renova automaticamente
drive_id não encontrado Config sem drive_id Consultar Graph API com token válido (Parte 3, Passo 4)
